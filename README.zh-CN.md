# UI Checker Skill

[English](README.md)

`ui-checker` 是一个给 Codex / agent 使用的前端 UI 质检 skill。它适合在没有设计师参与的情况下，帮助 agent 判断一个页面为什么看起来乱、不统一、粗糙或不够专业，并把这些问题转成可以执行的修复建议。

这个 skill 的核心原则是四件事：

- **一致性**：字体、颜色、按钮、图标、圆角、阴影、边框、组件结构要统一。
- **留白**：页面边距、模块间距、元素间距、卡片内边距和整体密度要舒服。
- **对齐**：文字、图标、按钮、表单、列表、表格和页面标题要对齐。
- **反馈**：hover、active、focus-visible、disabled、loading、empty、error、success、permission 等状态要完整。

## 什么时候用

当你希望 agent 做这些事情时，可以使用这个 skill：

- 检查一个 UI 是否显得混乱、不一致或没打磨完。
- 改善 AI 生成的前端页面，让它看起来不那么“草台”。
- 审查截图、页面实现、Tailwind class、CSS 或组件文件。
- 不要泛泛地说“还不错”或“需要优化”，而是指出具体问题。
- 输出有优先级的修复清单，并生成可直接交给另一个 coding agent 的 prompt。

示例 prompt：

```text
使用 ui-checker 检查这个页面，告诉我为什么它看起来有点乱。
```

```text
检查这个页面的字体、按钮、图标、间距和页面状态是否统一。
```

```text
帮我生成一个 UI 自检 prompt，让另一个 coding agent 可以直接修这个页面。
```

## 它会检查什么

这个 skill 会引导 agent 做一轮比较完整的 UI 审查，包括：

- 字体层级
- 按钮样式和状态
- 图标颜色和尺寸
- 背景、边框、圆角和阴影
- 间距、gap、对齐和页面密度
- 主色使用是否统一
- 输入框、表单、可点击反馈和基础可访问性
- 空状态、加载状态、错误状态、权限状态和成功反馈
- 下拉菜单、弹窗、卡片、导航、Tab、Badge 和表格
- 移动端、平板和桌面端响应式表现
- 文本溢出、无效数据、日期/数字格式和兜底文案
- 组件复用和 Tailwind token 使用纪律

## 预期输出

当用它审查 UI 时，skill 会要求 agent 按这个结构回答：

1. **UI 质量结论**：简短判断界面是干净、略乱，还是比较粗糙。
2. **主要问题**：列出最重要的问题、为什么影响 UI 质量、应该怎么修。
3. **修复优先级**：用 P0/P1/P2 区分必须马上修、应该修和可以后续优化的问题。
4. **可复制的修复 prompt**：生成一段可以直接交给 coding agent 的具体修复指令。

## 安装方式

把 `SKILL.md` 放到 Codex skills 目录下的 `ui-checker` 文件夹中。

示例：

```text
~/.codex/skills/ui-checker/SKILL.md
```

如果你的 agent 运行环境使用不同的 skills 目录，把同一个 `SKILL.md` 复制到对应位置即可。

## 文件说明

- `SKILL.md`：agent 实际读取和执行的 skill 指令。
- `README.md`：英文介绍和使用指南。
- `README.zh-CN.md`：中文介绍和使用指南。

## 设计目标

这个 skill 追求的是实用，不是替代设计师，也不是从零建立完整品牌设计系统。它的目标是帮助 agent 把日常前端页面做得更干净、更统一、更容易使用。
